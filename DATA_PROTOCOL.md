# ESP-NOW 通信数据帧协议规范

本文档专门针对发送端和接收端之间的通信数据格式、帧结构及解析注意事项进行说明。

## 1. 帧结构概述

发送端与接收端之间传输的有效载荷（Payload）长度固定为 **18 字节**，格式严格对齐。底层数据链路基于 ESP-NOW 协议传输。
每帧数据均以单字节 `0xAA` 作为帧头，以单字节 `0x55` 作为帧尾。

## 2. 数据结构定义

如果是在 C 语言中对该数据帧进行还原，对应的结构体定义如下（需注意添加1字节紧凑对齐 `__attribute__((packed))` 防编译器填充）：

```c
typedef struct __attribute__((packed)) {
    uint8_t header;        // 帧头 (固定 0xAA)
    uint8_t KEY[4];        // 4字节的按键状态位势
    int16_t rocker_l_;     // 左摇杆 X 轴 (大端序)
    int16_t rocker_l1;     // 左摇杆 Y 轴 (大端序)
    int16_t rocker_r_;     // 右摇杆 X 轴 (大端序)
    int16_t rocker_r1;     // 右摇杆 Y 轴 (大端序)
    int16_t dial;          // 拨轮数据 (大端序)
    uint8_t switch_left;   // 左侧拨动开关状态
    uint8_t switch_right;  // 右侧拨动开关状态
    uint8_t footer;        // 帧尾 (固定 0x55)
} remote_t;
```

## 3. 字段详解

具体字段在 18 字节数组中的分布如下：

| 偏移量 (Byte) | 字段名          | 数据类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `0` | `header` | `uint8_t` | 帧头标志，固定为 `0xAA` |
| `1~4` | `KEY[4]` | `uint8_t[4]` | 4字节按键状态位，表示面板上各独立按键的动作状态 |
| `5~6` | `rocker_l_` | `int16_t` | 左摇杆 X 轴坐标值 (大端序传输) |
| `7~8` | `rocker_l1` | `int16_t` | 左摇杆 Y 轴坐标值 (大端序传输) |
| `9~10` | `rocker_r_` | `int16_t` | 右摇杆 X 轴坐标值 (大端序传输) |
| `11~12` | `rocker_r1` | `int16_t` | 右摇杆 Y 轴坐标值 (大端序传输) |
| `13~14` | `dial` | `int16_t` | 拨轮旋转/模拟数据 (大端序传输) |
| `15` | `switch_left` | `uint8_t` | 左侧拨动开关状态 |
| `16` | `switch_right`| `uint8_t` | 右侧拨动开关状态 |
| `17` | `footer` | `uint8_t` | 帧尾标志，固定为 `0x55` |

### 3.1.1 `KEY[4]` 字段说明

`KEY[4]` 是 4 个字节的按键状态原始数据，用于承载面板上的独立按键或按键组状态。

当前工程里对它的处理方式是：

* 发送端 / 中继端直接按原始 4 字节透传，不在这一层做重新映射。
* 串口调试日志中以 4 个十六进制字节连续打印，便于观察按键状态变化。
* 由于本仓库里没有定义上游设备的逐位按键映射表，所以这里只能确认它是“4 字节按键状态位”，不能进一步推断每一位对应哪个物理按键。

如果后续拿到上游遥控器或采集板的具体按键定义，可以把 `KEY[4]` 继续细分成更明确的键位说明表。

## 4. 接收端解析注意事项

### 4.1 大端序(Big-Endian)转换
原生采集和传输的 16 位 (`int16_t`) 摇杆和拨轮数据均采用了 **大端序**。
由于作为接收端的 ESP32 主控默认是 **小端序 (Little-Endian)** 架构，因此当从字节流提取 `int16_t` 时，必须进行字节序的高低位互换才能获得正确的整型数值。

推荐的转换辅助函数如下：

```c
// 接收端解析大端序的参考函数
static inline int16_t be16_to_i16(const uint8_t *p) {
    return (int16_t)(((uint16_t)p[0] << 8) | (uint16_t)p[1]);
}

// 示例：如果 data 是接收到的 18 字节有效数据数组
// int16_t left_x = be16_to_i16(&data[5]);
// int16_t left_y = be16_to_i16(&data[7]);
```

### 4.2 数据完整性校验
在接收端 ESP-NOW 数据接收回调处理函数中，应首先验证长度定界符以过滤掉误传或截断的无效数据包：

```c
void recv_cb(const uint8_t *mac_addr, const uint8_t *data, int data_len) {
    // 强制判断长度 18 字节、帧头 0xAA、帧尾 0x55
    if (data_len == 18 && data[0] == 0xAA && data[17] == 0x55) {
        // 数据包合法，进行处理...
        // ...
    }
}
```
